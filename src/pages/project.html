TutorMatch

with Samuel Dovbenyuk

1. Names
Team: WebFusion
Members: Nevathan, Abishan, Adrian, Liyu
Client: Sam Dovbenyuk
Application: TutorMatch

2. Purpose
Original: A Tinder-style swipe-based tutor-tutee matching system.
Revised: Swipe functionality was removed due to a limited number of tutors. We now use an "accept" button for match confirmations. Tutors apply through an interview process to be featured on the site. This streamlines the matching system and meets client needs more effectively.

3. Data Description
Original: Included swipes and AI-based matching data.
Revised: Removed swipe and match-related tables. Added a 'notifications' table to enable engagement and timely updates.

Users: user_id, name, email, password, role, created_at
Tutors: tutor_id, user_id, GPA, hourly_rate, bio, subjects
Clients: client_id, user_id, preferred_subjects, budget
Notifications: notification_id, user_id, message, created_at, read_status

4. Functions Performed by the App
- Create, update, and delete profiles
- Tutors apply to be listed on the platform
- Tutees browse available tutors and accept matches
- Notifications sent to improve engagement
Removed: Swipe and AI matchmaking features

5. Technologies Used
Revised: PHP is used for server-side logic, data handling, and form processing. JavaScript enhances client-side interactivity. React is used for UI components. SQL manages the database. We removed Node.js to simplify deployment on our university-provided server.

6. Roles of Team Members
Revised: All members contributed to both front-end and back-end to ensure fair distribution and experience across all technologies.
- Nevathan: PHP backend logic, SQL queries for user authentication, React tutor acceptance flow, HTML structure edits, JavaScript event handling
- Liyu: Notification system using PHP/SQL, styled components using HTML/CSS, JavaScript for alert updates, React components
- Adrian: Front-end using React, SQL optimization for tutor search, PHP session management, HTML content layout, JavaScript for interactivity
- Abishan: HTML/CSS layout and responsive design, JavaScript logic for buttons and accept flow, PHP form validation, React component integration, SQL inserts for applications

7. Increments
Increment 1 (April 1): Basic MVP – User login, registration, and tutor application form
Increment 2 (April 8): Notifications, Accept Button logic, full tutee interface

8. Client Feedback and Changes
Sam commented, "The swipe system looks cool but might not work well with only a few tutors."
Change made: Replaced swipe with an accept button and simplified tutor-tutee matching.

Sam said, "Notifications would help keep users engaged."
Change made: We added a notification system to alert tutees about tutor responses and updates.

Sam also noted, "We don't need a separate tutor portal—just have tutors apply, and we approve them."
Change made: Removed the tutor-side of the site. Tutors now apply via form and are reviewed by admins.

Status: We haven’t shown the final version yet, but the feedback was incorporated into the revised direction. Sam remarked, "This is more realistic and clean – I think it'll work better for students here."
